# Schedular Bot

Телеграм-бот для управления личными задачами с напоминаниями, быстрым веб-поиском и погодой по городу.

## 🚀 Возможности

- **Управление задачами**
  - добавление задачи с датой/временем;
  - просмотр ближайшей задачи;
  - просмотр всех активных задач;
  - перенос времени задачи;
  - отметка задачи как выполненной.
- **Напоминания**
  - отложенные уведомления через Celery;
  - восстановление pending-задач после рестарта бота.
- **Погода**
  - получение текущей погоды через `wttr.in` (без API-ключа);
  - кэширование запросов к сервису погоды wttr.in;
  - сохранение данных на 10 минут в Redis;
  - сохранение города пользователя в БД;
  - перевод погодных описаний на русский;
  - реализован retry-механизм с exponential backoff для внешних API;
  - обработка сетевых ошибок и таймаутов.
- **Поиск**
  - поиск через DuckDuckGo (`ddgs`) прямо из бота.
- **Надёжность и поддержка**
  - централизованное логирование;
  - базовое покрытие тестами (`pytest`, `pytest-asyncio`, `pytest-cov`).

## ⚙️ Подготовка окружения

### 1) Клонирование

```bash
git clone <your_repo_url>
cd schedular_bot
```

### 2) Виртуальное окружение

```bash
python -m venv .venv
source .venv/bin/activate  # Linux/macOS
# .venv\Scripts\activate   # Windows (PowerShell)
```

### 3) Установка зависимостей

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### 4) Переменные окружения

Создайте `.env` в корне проекта:

```env
TELEGRAM_TOKEN=your_telegram_token_here
DATABASE_URL=postgresql://user:password@host:port/dbname
```

Можно использовать `.env.example` как шаблон.

## ▶️ Запуск

```bash
python -m app.main
```

При старте бот:

1. загружает `.env`;
2. поднимает соединение с PostgreSQL;
3. восстанавливает запланированные напоминания для активных задач;
4. начинает polling Telegram API.

## 🧠 Логика ключевых сценариев

### Добавление задачи

1. Пользователь выбирает «➕ Добавить задачу».
2. Вводит дату/время (например, `2026-02-10 18:30`, `сегодня 21:00`, `завтра 9:00`).
3. Вводит текст задачи.
4. Задача сохраняется в БД со статусом `pending`.
5. В `job_queue` создаётся отложенное напоминание.

### Напоминания

- По наступлению времени бот отправляет карточку задачи с кнопками:
  - «✅ Выполнена»
  - «⏰ Перенести»
  - «↩️ В меню»
- Перед отправкой бот дополнительно проверяет задачу в БД (актуальность времени и статус).

### Погода

- Бот получает город пользователя из БД.
- Если город сохранён — сразу возвращает погоду.
- Если нет — запрашивает новый город и сохраняет его после успешного ответа.

### Поиск

- Запрос пользователя валидируется.
- Поиск выполняется через DuckDuckGo.
- Бот отправляет до 5 результатов в формате `title + link`.

## 🗄️ Схема БД

Создаются таблицы:

- `tasks`
  - `id TEXT PRIMARY KEY`
  - `user_id BIGINT NOT NULL`
  - `title TEXT NOT NULL`
  - `scheduled_time TIMESTAMPTZ NOT NULL`
  - `status TEXT NOT NULL` (`pending` / `done`)
- `users`
  - `user_id BIGINT PRIMARY KEY`
  - `city TEXT`

## 🧹 Линтинг и проверка стиля

Для поддержания чистоты кода и соблюдения стандартов используется **[Ruff](https://github.com/charliermarsh/ruff)**.

### Установка

```bash
pip install ruff
```

### Проверка кода без исправлений

```bash
ruff check .
```
Запускает линтер на всех файлах проекта.
Отображает потенциальные ошибки, нарушения стиля и предупреждения.


### Автоматическое исправление форматирования

```bash
ruff format .
```
Исправляет пробелы, отступы, переносы строк и другие мелкие несоответствия стайлгайду.

Рекомендуется запускать перед коммитом или перед созданием Pull Request.
Ruff интегрируется с pyproject.toml для настройки правил линтинга и игнорирования отдельных ошибок.
